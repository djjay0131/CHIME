# Memory Bank: CHIME Documentation System

## Overview

This Memory Bank is the continuity layer for CHIME work. Read it after context resets and update it when understanding of the codebase changes.

## Core Files

- `projectbrief.md`: project scope and documentation mission
- `productContext.md`: why this repository exists and who uses it
- `systemPatterns.md`: architecture and flow between code layers
- `techContext.md`: build, environment, and tooling constraints
- `phases.md`: documentation phases and coordination state
- `activeContext.md`: current focus and immediate next steps
- `progress.md`: what is documented and what is still unclear
- `architecturalDecisions.md`: key documentation and organization decisions

## Working Rule

Before making substantial documentation changes, read:

1. `projectbrief.md`
2. `systemPatterns.md`
3. `techContext.md`
4. `activeContext.md`
5. `progress.md`

Update `activeContext.md` and `progress.md` when priorities or understanding change.
