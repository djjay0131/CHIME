# Documentation Acceptance

Documentation changes in this repository should satisfy these checks:

- A new reader can identify where build configuration lives.
- A new reader can tell which files are core runtime code versus experiment orchestration.
- At least one representative experiment script is explained end to end.
- The docs distinguish compile-time parameters from runtime parameters.
- The docs point to the benchmark entrypoint used by the experiment scripts.
- The docs avoid claiming support or portability beyond what the repository README already states.
