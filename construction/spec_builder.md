# Documentation Builder

Use this template when a documentation request is vague.

## Questions To Answer

1. Which audience is this for: paper reproducer, code maintainer, or new contributor?
2. Which layer needs explanation: setup, C++ runtime, benchmark harness, or experiment scripts?
3. What output is needed: README update, focused guide, or memory-bank refresh?
4. How will we verify the docs are good enough?

## Default Execution Order

1. Read the relevant source files.
2. Trace the entrypoint to the implementation.
3. Identify compile-time and runtime configuration points.
4. Write the shortest document that closes the knowledge gap.
5. Link it from an existing README so it is discoverable.
