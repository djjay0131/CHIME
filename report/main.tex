\begin{abstract}
This report documents the reproduction of experiments from the CHIME paper (SOSP '24), which proposes a hybrid B+ tree with hopscotch hashing for disaggregated memory. We reproduced the key performance results on 10 CloudLab r650 nodes with 100~Gbps RDMA, comparing CHIME against four baselines (Sherman, SMART, ROLEX, and SMART-SC). We present throughput-latency curves (Figure~12), cache consumption analysis (Figure~14), and factor analysis (Figures~15a/15b) from the original paper, along with observations on reproducibility and performance differences.
\end{abstract}

\section{Introduction}

CHIME~\cite{chime2024} is a cache-efficient and high-performance hybrid index for disaggregated memory (DM). It combines B+ tree internal nodes with hopscotch-hashing-based leaf nodes to simultaneously achieve low cache consumption and low read amplifications. The paper introduces three key techniques: (1) three-level optimistic synchronization, (2) access-aggregated metadata management, and (3) hotness-aware speculative read.

This report documents our effort to reproduce the paper's main experimental results on CloudLab infrastructure as part of the CS~6204 course project.

\subsection{Scope}

We reproduced the following experiments from the original paper:
\begin{itemize}
    \item \textbf{Figure 12:} YCSB throughput-latency comparison across 6 workloads (LOAD, A--E) with 5 methods
    \item \textbf{Figure 14:} Cache consumption comparison across varying dataset sizes
    \item \textbf{Figure 15a:} Factor analysis starting from Sherman (B+ tree baseline)
    \item \textbf{Figure 15b:} Factor analysis starting from ROLEX (learned index baseline)
\end{itemize}

\section{Experimental Setup}

\subsection{Hardware}

All experiments were conducted on CloudLab r650 nodes:
\begin{itemize}
    \item \textbf{CPU:} 2$\times$ Intel Xeon (36 cores each)
    \item \textbf{Memory:} 256~GB DDR4
    \item \textbf{NIC:} Mellanox ConnectX-6 (100~Gbps)
    \item \textbf{Network:} 100~Gbps Ethernet
    \item \textbf{Cluster:} 10 nodes total
\end{itemize}

\subsection{Configuration}

Following the paper's experimental setup (Section~5.1):
\begin{itemize}
    \item Compute nodes (CNs): 4~GB local cache, 64 cores (1 client per core)
    \item Memory node (MN): 64~GB DRAM, 1 CPU core
    \item YCSB workloads: 60M entries, 8-byte keys, 8-byte values, Zipfian distribution
\end{itemize}

\subsection{Software}

\begin{itemize}
    \item CHIME artifact: forked from \texttt{dmemsys/CHIME}
    \item SMART baseline: \texttt{dmemsys/SMART}
    \item ROLEX baseline: \texttt{River861/ROLEX} (CHIME authors' reimplementation)
    \item Sherman baseline: built from CHIME repo with different CMake flags
    \item Mellanox OFED drivers, memcached for metadata coordination
\end{itemize}
